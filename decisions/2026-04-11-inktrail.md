# InkTrail — Bootstrap Decisions

## Project Profile

- **Name**: InkTrail — AI-paced text delivery for Google Docs
- **Type**: Full-stack TypeScript web app
- **Structure**: Single package (not a monorepo)
- **Size**: Small (~10 source modules, early development)
- **UI**: React 19, Tailwind CSS v4
- **Framework**: Hono (server), Vite (frontend bundler), tRPC v11
- **DB**: Drizzle ORM + PostgreSQL (via Render free-tier Postgres)
- **Runtime**: Node.js via tsx (dev + prod)
- **Deployment**: Render.com free tier
- **Team**: Sole developer

## Tooling Decisions

### Formatter: Biome

**Why**: Zero-config, fast (<10ms), handles TS/TSX/JSON. Greenfield project with no existing formatter. AgentiCNC (same React + tRPC + Drizzle stack) made the same call. Biome's unified format+lint eliminates maintaining two separate tools.

**Considered**:
- oxfmt — faster (~0.1ms) but format-only; would require a separate linter
- Prettier — too slow (~200ms) for PostToolUse hooks; more config required

**Agentic fit**: Excellent — fast enough for PostToolUse on every edit.

### Linter: Biome

**Why**: Same `biome check` command — zero additional config. No React/Next.js-specific ESLint plugins needed for this stack.

**Considered**:
- oxlint — would pair with oxfmt, but adds a second tool when Biome covers both
- ESLint — no compelling plugin requirements; config overhead not worth it

**Agentic fit**: Excellent — unified with formatting, single fast pass.

### Test Framework: Vitest (aspirational — not yet set up)

**Why**: No existing test framework. TypeScript-first, native ESM, fast. Installed and scripts added; no tests written yet.

**What's needed**: First test should cover the worker's pacing logic (most critical behavioral code).

**Agentic fit**: Excellent — fast watch mode for TDD.

### Git Hooks: PostToolUse + PreToolUse only (no hook runner)

**Why**: Sole developer using Claude Code. PostToolUse hooks (file size + Biome) catch issues at edit time. PreToolUse before `git commit` runs `tsc --noEmit && biome check`.

**Considered**: Lefthook, Husky — rejected; sole developer doesn't need a separate hook runner.

**Agentic fit**: Excellent for sole developer.

### Build: Vite + tsx (kept as configured)

Vite for frontend, tsx for dev/prod server. No change. Note: `npm run build` only builds the frontend — server runs via `tsx` directly on Render.

### Package Manager: npm (kept as configured)

npm with `package-lock.json`. No change.
